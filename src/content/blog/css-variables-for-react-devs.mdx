For years, React developers have relied on `ThemeProvider` context to pass design tokens down the component tree. While this works, it adds runtime overhead and complexity. Enter CSS Variables (Custom Properties).

## The Power of Scope

CSS variables observe the DOM structure, meaning you can override values locally without complex context overrides.

```css
:root {
  --primary: #blue;
}

[data-theme='dark'] {
  --primary: #lightblue;
}

.card {
  /* Uses whatever --primary is currently set to in this scope */
  border: 1px solid var(--primary);
}
```

## Integrating with React

You can set CSS variables directly in the `style` prop of your React components for dynamic values:

```jsx
function ProgressBar({ progress }) {
  return (
    <div 
      className="progress-bar"
      style={{ '--progress': `${progress}%` }} 
    />
  );
}
```

```css
.progress-bar {
  width: var(--progress);
  background: var(--primary);
  transition: width 0.3s ease;
}
```

This approach keeps your Javascript light and lets the browser's optimized transform engine handle the heavy lifting.
